# 여수 섬 해양쓰레기 유입 예측 시스템
### 여수 섬 박람회 데이터 공모전 출품 기획서 초안

---

## 1. 개요

본 프로젝트는 **여수 인근 섬 지역의 해양쓰레기 유입량을 사전 예측**하는 Grey-box 하이브리드 모델을 개발하는 것을 목표로 한다. 단순한 통계적 예측을 넘어, 해양 물리 시뮬레이션(MOHID)과 딥러닝(GNN/LSTM)을 결합하여 **"왜 이 섬에 쓰레기가 모이는가"** 라는 인과적 근거를 제시할 수 있는 설명 가능한 예측 체계를 구축한다.

---

## 2. 문제 정의

### 2.1 해양쓰레기는 '움직이는' 문제다

해양쓰레기는 수거 시점의 정적인 통계 데이터만으로는 설명할 수 없다. 바람과 조류에 의해 지속적으로 이동하는 **동적 부유물**이기 때문에, 어디에서 발생해서 어느 섬으로 유입되었는지 추적하려면 해양 유동 정보가 반드시 필요하다.

### 2.2 여수는 데이터가 부족하다

여수 섬 지역의 실측 해양쓰레기 관측 데이터는 현재 **4개 지점에 불과**하다. 이는 딥러닝 모델을 여수에 직접 적용하기에 턱없이 부족한 양이다.

### 2.3 기존 블랙박스(Black-box) AI의 한계

데이터가 부족한 상황에서 일반적인 머신러닝 모델을 적용할 경우 두 가지 문제가 발생한다.

- **과적합(Overfitting)**: 소수의 관측치에 과도하게 맞춰져 일반화 성능이 떨어진다.
- **설명 불가능**: "왜 이 섬에 쓰레기가 많이 유입되는가"라는 인과 관계를 설명하지 못한다.

이를 해결하기 위해 본 프로젝트는 **물리 모델이 제공하는 사전 지식**을 AI에게 부여하는 Grey-box 접근법을 채택한다.

---

## 3. 핵심 전략: Grey-box 하이브리드 모델링

### 3.1 전체 연구 흐름

```
[Step 1] 남해안 전체로 연구 범위 확장
         → 남해 35개 섬 기준 데이터로 물리·AI 모델 구축 및 검증

[Step 2] MOHID 물리 시뮬레이션
         → 어항·하천 입구를 방출점으로 설정, 해수유동·조위·풍속 데이터로
           쓰레기 입자의 이동 궤적 시뮬레이션

[Step 3] 물리 시뮬레이션 결과 검증
         → MOHID 예측 입자 유입량 vs. 실제 남해 섬 쓰레기 통계 비교
           → 물리 모델의 예측력(입자 도달 여부) 평가

[Step 4] GNN/LSTM 기반 딥러닝 고도화
         → 섬(노드) + 해류(엣지) 그래프 구조로 공간적 흐름 학습
         → 남해 데이터로 사전학습(Pre-training)

[Step 5] 여수 전이 적용 (최종 목표)
         → 남해에서 학습한 '유입 법칙'을 여수 섬 데이터로 미세조정(Fine-tuning)
         → 여수 356개 섬 대상 해양쓰레기 유입 예측
```

### 3.2 물리 모델: MOHID 수치 시뮬레이션

**MOHID(Marine and Estuarine Simulation System)** 는 해수유동을 수치적으로 모사하는 물리 모델이다. 본 프로젝트에서는 다음 역할을 한다.

- **입력**: 해수유동(U·V 벡터, 코페르니쿠스), 조위·조수, 풍속·풍향
- **방출점 설정**: 전남 연안 어항구역 및 하천-해안 연결 입구 좌표를 QGIS로 추출하여 쓰레기 입자 가상 방출
- **출력**: 시간대별 입자 이동 궤적 및 특정 섬 반경 내 유입 입자 수 (`mohid_particle_count`)

이 `mohid_particle_count`가 AI 모델의 핵심 입력 피처(Feature)가 된다. AI는 단순히 과거 쓰레기 수거량이 아니라 **"물리적으로 얼마나 많은 입자가 그 섬으로 향했는가"** 라는 해양 역학 정보를 학습하게 된다.

### 3.3 딥러닝 모델: GNN + LSTM

| 구성요소 | 역할 |
|---|---|
| **GNN (그래프 신경망)** | 섬을 노드(Node), 해류 방향을 엣지(Edge)로 연결하여 섬 간 쓰레기 전파 관계(공간 의존성)를 학습 |
| **LSTM (장단기 메모리)** | 계절·조류 변화 등 시간적 패턴을 학습하여 순서 있는 유입 예측 |
| **물리 피처 결합** | MOHID 입자 수를 주요 입력으로 사용하여 Grey-box 구조 실현 |

### 3.4 공간 전이학습 (Spatial Transfer Learning)

데이터 부족 문제를 해결하기 위한 핵심 전략이다.

1. **사전학습(Pre-training)**: 남해안 35개 섬 데이터(상대적으로 풍부)로 '해양쓰레기 유입의 일반 법칙'을 모델에 학습시킨다.
2. **미세조정(Fine-tuning)**: 여수 섬(4개 관측점)의 소수 데이터로 여수 특화 패턴을 추가 학습한다.

이를 통해 여수 현지 관측 데이터가 극히 적더라도 신뢰할 수 있는 예측이 가능하다.

## 4. 데이터 현황

### 4.1 확보 완료 데이터

| 데이터 | 출처 | 내용 |
|---|---|---|
| 해수유동 데이터 (U·V 벡터) | 코페르니쿠스(Copernicus) | 남해 2025년 전체 및 2026년 상반기 분기별 데이터 |
| 조위·기상 통합 데이터 | 기상청·해양수산부 | 남해 관측소 시간별 조위·풍속·풍향 데이터 |
| 남해 해양쓰레기 통계 | 공공데이터 | 남해 35개 섬 수거 실적 (지역명·수량·무게·위경도) |
| 전남 하천·어항 입구 좌표 | QGIS 추출 | 쓰레기 방출점 위경도 (전남 연안) |
| 여수 섬 매핑 데이터 | 전라남도 공간정보 플랫폼 | 여수 섬 위경도(WGS84), MOHID 그리드 매핑 정보 |
| MOHID 입자 시뮬레이션 결과 | MOHID 실행 | 남해 35개 섬 기준 입자 도달 수 (2025년 1분기 완료) |

### 4.2 AI 학습 데이터셋 (`AI_Training_Dataset.csv`)

MOHID 입자 카운트와 실제 쓰레기 수거량을 결합한 학습용 데이터셋이 구성되어 있다.

| 컬럼 | 설명 |
|---|---|
| `지역명` | 남해 관측 지역 이름 |
| `mohid_particle_count` | MOHID 시뮬레이션 상 해당 지역 2km 반경 내 도달 입자 수 |
| `수량(개)` | 실제 해양쓰레기 수거 수량 |
| `무게(kg)` | 실제 해양쓰레기 수거 무게 |

> **현재 한계**: MOHID 시뮬레이션은 2025년 1분기 데이터 기준으로 구동 완료되었으며, 입자 수(`mohid_particle_count`)가 실제 연간 수거 통계와 시간 범위가 불일치하는 상태다. 전체 연간 데이터로의 확장이 진행 중이다.

| 여수 전이학습 적용 | ⏳ 예정 | 남해 학습 모델 → 여수 섬 미세조정 |

## 6. 기대 효과 및 활용 방안

### 6.1 실무 밀착형 조기경보 서비스

본 시스템의 최종 목표는 기상·해양 예보 데이터를 실시간으로 반영하여 **당일~익일 단위의 섬별 해양쓰레기 유입 예측 알림**을 제공하는 것이다.

> 예시: *"내일 오전 금오도 동측 해안에 해양쓰레기 유입이 예상됩니다. 해당 구역 수거반 사전 배치를 권장합니다."*

### 6.2 청소선 최적 경로 제안

예측된 섬별 유입량 데이터를 기반으로, 수거 우선순위와 청소선의 최적 순회 경로를 자동으로 산출할 수 있다. 이는 수거 인력과 선박 운항 비용의 효율적 배분에 기여한다.

### 6.3 정책·연구 기여

- 연안 해양쓰레기 발생원(하천 유출·어항) 위치와 유입 경로의 가시화
- 계절별·기상 조건별 유입 패턴 분석을 통한 수거 시기 최적화 근거 제공
- 향후 여수 외 타 연안 지자체로의 모델 확장 가능성

## 8. 미결 사항 및 확인 필요 내용

> 아래 항목은 기획서 확정 전 팀 내 확인이 필요한 사항입니다.

1. **공모전 제출 요건**: 제출 형식(보고서·발표·코드)과 분량 기준 확인 필요.
2. **MOHID 전체 연간 실행 일정**: 2025년 연간 데이터 시뮬레이션 완료 시점 확정 필요 (현재 1분기만 완료).
3. **남해 쓰레기 통계 데이터 출처 명시**: `namhae_wide_marine_debris_windows.csv`의 공식 출처(기관명, 수집 기간 등) 기획서 내 명확히 기재 필요.
4. **GNN 고도화 완료 범위**: 공모전 제출 시점까지 GNN/LSTM 모델 중 실제 결과를 보여줄 수 있는 범위 정의 필요.
5. **여수 4개 관측점 데이터 상세**: 관측 지점명, 데이터 수집 기간, 수거량 규모 등 구체적인 내용 추가 필요.
