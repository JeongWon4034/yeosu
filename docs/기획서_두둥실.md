## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 두둥실 (DuDungSil) |
| **한 줄 요약** | 가장 먼저, 가장 효과적으로 대응하게 만드는 예측 기반 의사결정 시스템 |
| **핵심 모델** | MOHID + GNN + LSTM 하이브리드 Grey-box 모델 |
| **대상 지역** | 전라남도 여수시 다도해 (335개 섬) |
| **예측 범위** | 향후 6~24시간 내 특정 섬 쓰레기 유입 확률 산출 |
| **주요 산출물** | 실시간 유입 경로 시각화 대시보드, 정화 인력·선박 배치 권장 리포트 |

### 1.1 프로젝트 배경

> **"국내 연안 해양쓰레기 분야에서 MOHID + GNN을 결합한 사례는 확인되지 않으며,
> 본 아이디어는 해당 구조의 선도적 시도입니다."**

해양쓰레기 관리는 전국적으로 연간 수천억 원의 예산이 투입되는 국가적 과제임에도,
현재 대응 방식은 '발생 후 수거'라는 사후 대응에 머물러 있습니다.
**두둥실**은 물리 시뮬레이션(MOHID)과 딥러닝(GNN + LSTM)을 결합하여,
쓰레기가 어디서 출발해 어느 섬에 언제 도달하는지를 **사전에, 설명 가능한 방식으로** 예측합니다.

## 2. 문제 정의 및 제안 배경

### 2.1 현황 — 심각한 해양쓰레기 문제

```
여수시 해양쓰레기 현존량:  3,675톤  (전남 2위)
연간 수거량:               약 3,000톤
연간 투입 예산:            30억 원 이상
부정적 영향 영역:          어업 피해 / 선박 운항 위험 / 관광 이미지 훼손
```

> **[권장 이미지 ①]** 여수시 해양쓰레기 수거 현황 통계 그래프
> (연도별 수거량 추이, 예산 투입 현황 막대/꺾은선 차트)

### 2.2 핵심 문제 — 사후 대응의 구조적 비효율

| 구분 | 현재 방식 | 문제점 |
|------|-----------|--------|
| **대응 시점** | 쓰레기 발견 후 수거 | 선제적 차단 불가 |
| **자원 배분** | 전 구역 균등 배치 | 예산·인력 비효율 |
| **의사결정** | 담당자 경험 의존 | 데이터 기반 근거 부재 |
| **예측 도구** | 없음 | Hotspot 사전 파악 불가 |

해양쓰레기 발생량은 **강수·해류·바람**에 따라 급변하며, 집적 구간(Hotspot)이
**특정 시점, 특정 지역**에 집중 발생합니다. 그러나 현재는 이를 사전에 예측할 수 없어
전 구역 균등 대응이라는 비효율적 방식이 지속되고 있습니다.

### 2.3 데이터 활용 적합성

두둥실은 **이미 확보된 실증 데이터**를 기반으로 개발되고 있습니다:

| 데이터 유형 | 규모 | 출처 |
|-------------|------|------|
| 해수유동 (u/v 성분) | 6개 분기 파일, 시간별 | 국립해양조사원 |
| 조위 (관측/예측) | 약 7,000개 시간 레코드 | 조위관측소 |
| 풍속·풍향 | 8개 기상관측소, 시간별 | 기상청 |
| 여수 실측 수거 데이터 | 4개 지점 | 현장 수거 기록 |
| 남해안 수거 데이터 | 32개 관측소 | 해양환경공단 |
| 방출점 (오염원) | 127개소 | QGIS 공간분석 도출 |
| 대상 섬 위경도 | 335개 | 국가공간정보포털 |

## 4. 독창성 및 차별성

### 4.1 Grey-box 하이브리드 — 설명 가능한 예측

| 구분 | 블랙박스 AI | 두둥실 (Grey-box) |
|------|------------|-------------------|
| **예측 근거** | 없음 (결과만 제공) | 해류·풍속 기반 물리적 경로 |
| **현장 신뢰도** | 낮음 | 높음 (원인 설명 가능) |
| **이상 기상 대응** | 불안정 | 물리 모델로 안정적 |
| **구성 요소** | 단일 모델 | MOHID + GNN + LSTM |

> **[권장 이미지 ③]** Grey-box 모델 구조 다이어그램
> — MOHID(물리) → GNN(공간) → LSTM(시간) → 위험도 산출의 플로우차트

### 4.2 공간적 연결성 반영 — 그래프 기반 해양 모델링

```
기존: 특정 지점 단위 독립 예측 (섬 ↔ 섬 관계 무시)
두둥실: 섬과 섬을 해류 흐름 기반으로 연결된 그래프로 구성
        → 쓰레기 이동 및 확산 경로를 네트워크 형태로 학습
```

- **노드(Node)**: 섬(해안 세그먼트) 23개 + 방출점 127개 = **총 150개**
- **엣지(Edge)**: 해류 방향 기반 이동 확률 (거리 80km 임계값)
- **엣지 가중치**: 해류 방향 도트곱 + source_count 비율로 결정

> **[권장 이미지 ④]** `output/step2_graph.png`
> — 남해안 섬 그래프 네트워크 시각화 (노드: 섬, 엣지: 해류 연결 경로)

### 4.3 전이학습으로 데이터 부족 극복

```
문제: 여수 실측 데이터 4개 지점 → 딥러닝 학습에 극도로 부족

해결:
  [1단계 Pre-training]  남해안 32개 관측소 데이터로 일반 패턴 학습
  [2단계 Fine-tuning]   여수 지형·박람회 개최·하천/항만 거리 등 반영
```

| 구분 | 데이터 규모 | 학습 목표 |
|------|------------|-----------|
| **Pre-training** | 남해안 32개 지점 | 일반적 해양쓰레기 유입 패턴 |
| **Fine-tuning** | 여수 4개 지점 | 여수 지역 특화 맥락 반영 |

## 6. 데이터 파이프라인

### 6.1 데이터 수집 및 통합 (`preprocess.py`)

1. **MOHID 결과 + 실측 레이블 병합** (inner join, 23개 섬)
2. **환경 변수 시계열 통합** — 조위, 기상, 해류를 시간 단위로 정합
3. **결측치 처리** — 선형 보간(Linear Interpolation)
4. **방출점 인코딩** — 해변/항만/하천/어업 4개 유형 One-hot 인코딩
5. **출력**: 3개 CSV 파일 (`island_merged`, `env_timeseries_merged`, `release_points_encoded`)

### 6.2 그래프 구성 (`build_graph.py`)

```python
# 노드 특성: [위도, 경도, is_beach, is_port, is_river, is_fishery]
node_features: shape [150, 6]

# 엣지 조건: 방출점→섬 거리 < 80km, 해류 방향 가중치 반영
edge_features: [거리, u_current, v_current, 속도, source_count_ratio]

# 엣지 가중치: 해류 방향 도트곱 × source_count 비율
```

> **[권장 이미지 ⑤]** 그래프 구성 개념도
> — 방출점(하천·항만·해변) → 섬 간 연결을 보여주는 개념 다이어그램
> (해류 벡터 방향 화살표, 섬 노드, 연결 엣지 표현)

### 6.3 시계열 시퀀스 준비 (`prepare_lstm.py`)

```
슬라이딩 윈도우:
  - 윈도우 크기: 168시간 (7일)
  - 스텝: 24시간 (1일)
  - 특성: 조위, 조위예측, 풍속, 풍향, u_wind, v_wind, u_current, v_current
  - 결과: ~453개 시퀀스 (shape: [453, 168, 8])
```

## 7. AI 모델 상세 설계

### 7.1 LSTMEncoder — 시간 패턴 학습

```
입력:  [batch, 168, 8]  (7일 × 8개 환경변수)
           ↓
      LSTM 2층 (64 hidden units, dropout 0.2)
           ↓
      마지막 은닉 상태 추출
           ↓
      Linear + ReLU → embed_dim=32
           ↓
출력:  [batch, 32]  (시간 임베딩)
```

학습 목표: 조위·풍속·해류의 시간적 변화 패턴 → 언제 쓰레기가 이동하는가

### 7.2 MarineDebrisGNN — 공간·시간 통합 예측

```
입력:
  - 그래프 (노드 특성 [150, 6], 엣지 인덱스, 엣지 특성 [E, 5])
  - 환경 시퀀스 [1, 168, 8]
           ↓
  [1] LSTM → 시간 임베딩 [1, 32] → 모든 노드로 확장 [150, 32]
           ↓
  [2] 노드 특성 + 시간 임베딩 Concatenate → [150, 38]
           ↓
  [3] GAT Layer 1: 38 → 64 × 4 heads = 256 (edge_dim=5)
  [4] ELU + Dropout(0.3)
  [5] GAT Layer 2: 256 → 32 × 1 head
           ↓
  [6] Linear → 1 (노드별 스칼라 예측)
           ↓
출력:  [150]  (각 노드의 쓰레기 유입량 예측)
```

> **[권장 이미지 ⑥]** `output/pred_vs_actual.png`
> — LOOCV 예측값 vs 실측값 산점도 (모델 성능 시각화)

> **[권장 이미지 ⑦]** `output/training_loss.png`
> — 훈련 손실 곡선 (학습 수렴 과정 시각화)

### 7.3 손실 함수 (물리적 제약 반영)

```
L = MSE(예측값, 실측값) + λ × ReLU(예측값 - SC_MAX)

  - MSE: 표준 회귀 손실
  - λ = 0.01: 물리적 제약 가중치
  - SC_MAX = 34,385: MOHID 시뮬레이션 최대 입자수 (물리적 상한선)
```

물리 모델의 한계를 초과하는 예측에 패널티를 부여하여 **물리적으로 타당한** 예측을 유도합니다.

### 7.4 학습 전략 — Leave-One-Out Cross-Validation (LOOCV)

데이터 희소성(23개 레이블) 문제를 대응하는 엄격한 검증 방식:

```
총 23개 섬 데이터:
  - 1개 섬을 테스트 세트로 제외
  - 나머지 22개로 학습
  - 23회 반복 → 모든 섬에 대한 예측 성능 평가
```

평가 지표: **MAE, RMSE, R², Pearson 상관계수**

| **Detailed Report** | 섬별 위험도 카드, 팝업 정보 (수거량, 좌표 등) |

### 8.2 인터랙티브 지도 기능

- **PyDeck 기반** 지리 공간 시각화
- 여수 경계 GeoJSON 오버레이
- **색상 코딩** 위험도 표시 (녹색 → 노란색 → 빨간색)
- 섬 팝업: 이름, 수거량, 무게, 좌표 정보

### 8.3 사이드바 컨트롤

- 날짜 범위 슬라이더
- 쓰레기 유형 필터 (부유/해안/침적/어구류/유류)
- 섬 선택 멀티셀렉트
- 위험도 임계값 조정

## 9. 개발 현황 및 단계별 로드맵

### 9.1 완료 단계

| 항목 | 내용 |
|------|------|
| **시공간 데이터 파이프라인** | QGIS 기반 해안선 세그먼트 분할, 공간 분석 기반 마련 |
| **물리 시뮬레이션** | MOHID Lagrangian 입자 추적 (2023~2025년 데이터, 12개 시기, 1시간 간격) |
| **AI 학습용 피처 추출** | 섬 반경 1km 버퍼와 입자 궤적 공간 중첩 → 정량적 유입량 산정 |
| **마스터 데이터셋 구축** | 남해안 32개 관측소 + 물리 모델 결과 융합 통합 데이터셋 |
| **전이학습 사전학습** | GNN/LSTM 모델에 남해안 해양쓰레기 유입 패턴 선행 학습 완료 |
| **모델 학습 구조 구현** | MLP → GNN + LSTM 하이브리드 구조 구현 및 LOOCV 검증 |
| **대시보드 프로토타입** | Streamlit 기반 5탭 대시보드 (지도, 예측, 경로, 분석, 리포트) |

### 9.2 진행 중

| 항목 | 내용 |
|------|------|
| **Fine-tuning 고도화** | 여수 지역 특화 맥락 주입 (박람회, 산업단지, 하천 거리 등) |
| **예측 정확도 개선** | LOOCV 검증 결과 기반 하이퍼파라미터 튜닝 |
| **대시보드 연결** | 학습된 모델과 대시보드 실시간 연결 |

### 9.3 예정 (확장 로드맵)

```
[단기] 여수 특화 Fine-tuning 완성 → 여수 335개 섬 전수 예측
[중기] 자동 리포트 발송 시스템 (매일 아침 RPA 자동화)
       "내일자 1순위 정화 대상: 금오도 동측 (유입 위험도 85%)"
[중기] 청소선 GPS·배차 시스템 API 연동 → 최적 순회 라우팅 완전 자동화
[장기] 전국 다도해 표준 플랫폼 확장 (신안군, 통영시 등)
[장기] UAV/위성 SAR 드론 투입 + Vision AI 교차검증 + Self-learning 루프
```

### 10.1 행정 효율화 및 비용 절감

| 지표 | 현재 | 두둥실 적용 후 (예상) |

| 수거 효율 | 기준치 | 향상 |

- 집적 구간 파악으로 불필요한 순찰·운항 감소 및 수거 효율 향상

### 10.2 2026 여수 세계섬박람회 지원

> **"2026 여수 세계섬박람회 개최 연도를 정적 특성 변수로 반영하여 우선순위 산출"**

- 관광객 방문이 잦은 핵심 섬들의 미관 선제 관리
- '청정 해양 관광도시 여수'의 글로벌 브랜드 가치 제고
- 박람회 기간 집중 모니터링 및 실시간 대응 체계

### 10.3 모델의 높은 확장성

```
여수 (4개 지점) → 전이학습 → 신안군, 통영시 등 전국 연안 지자체
                              데이터 희소 지역에서도 즉시 도입 가능
```

- 전이학습 구조 모듈화 → **해양쓰레기 조기경보 표준 플랫폼** 가능성
- 다중 매체 교차 검증 (드론 + SAR + Vision AI) 루프 추가로 자가 학습 진화

### 10.4 학술·기술적 기여

> **"국내 연안 해양쓰레기 분야에서 MOHID + GNN 결합의 선도적 시도"**

- 해양 물리 모델과 그래프 신경망의 융합 → 새로운 방법론 제시
- 어텐션 가중치 시각화로 **어떤 오염원이 어느 섬에 영향을 주는지** 설명 가능
- 데이터 희소 환경에서의 전이학습 적용 사례 학술 기여
